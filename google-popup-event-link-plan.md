# 아임웹 팝업-이벤트 연동 기획안

## 대상 예시

- 팝업: `https://www.beauskin.co.kr/`
- 이벤트 게시글: `https://www.beauskin.co.kr/51`
- 케이스: 구글 이벤트 참여방법 안내

## 목적

- 팝업과 이벤트 게시글을 하나의 운영 단위로 묶어서 관리한다.
- 게시글은 수동 등록하되, 팝업 만료 시점에 맞춰 이벤트 게시글도 자동으로 종료 상태처럼 보이게 한다.
- 공지사항은 종료 처리 대상에서 제외하고, 이벤트 게시글만 관리 대상으로 둔다.

## 전제

- 아임웹 기본 관리자 기능만으로 게시글 상태를 실제로 자동 변경하는 것은 기대하지 않는다.
- 따라서 이 문서는 `헤더 코드에 넣는 스크립트`로 화면상 종료 처리처럼 보이게 만드는 방안을 기준으로 한다.
- 실제 DB의 게시글 비공개 전환, 삭제, 이동은 별도 서버 API나 관리자 자동화가 있어야 한다.

## 운영 원칙

1. 팝업 1개는 이벤트 게시글 1개와 1:1로 연결한다.
2. 이벤트 게시글은 수동으로 작성한다.
3. 팝업 종료일이 되면 이벤트 게시글도 동일한 종료일 기준으로 종료 처리한다.
4. 공지사항은 종료 대상에서 제외한다.
5. 이벤트 게시글은 종료 후에도 URL 접근은 가능하되, 실제 신청/참여 동작은 막는다.

## 분류 규칙

- 이벤트 게시판 대상
  - 참여, 신청, 응모, 혜택, 할인, 사은품, 쿠폰, 기간 한정 안내
  - 예시처럼 `구글 이벤트 참여방법` 같은 참여형 안내
- 공지사항 대상
  - 진료 안내, 휴진 안내, 운영 시간 안내, 이용 안내, 병원 공지
  - 단순 정보 전달 목적의 고지

## 연결 규칙

- 팝업 제목과 이벤트 게시글 제목에 공통 식별자를 넣는다.
- 추천 형식:
  - 팝업 제목: `[EVT-20260609-01] 구글 이벤트 참여방법`
  - 이벤트 제목: `[EVT-20260609-01] 구글 이벤트 참여방법`
- 관리자는 이 식별자를 기준으로 같은 건인지 확인한다.
- 종료일도 동일한 식별자에 묶어서 관리한다.

## 추천 데이터 항목

- 관리 ID
- 팝업 URL
- 이벤트 게시글 URL
- 시작일
- 종료일
- 게시판 종류
- 상태값: 예정, 진행중, 종료예정, 종료완료
- 담당자

## 화면상 종료 처리 방식

- 이벤트 게시판 목록에서 종료된 글은 숨김 처리한다.
- 상세페이지에는 종료 안내 문구를 노출한다.
- 신청 버튼, 링크 버튼, CTA는 비활성화한다.
- 직접 URL로 들어온 사용자는 목록 또는 안내 영역으로 유도한다.

## 스크립트 동작 원칙

- 페이지가 열릴 때 현재 날짜를 기준으로 종료 여부를 판단한다.
- 이벤트 전용 게시글에만 적용한다.
- 공지사항 게시판에는 아무 작업도 하지 않는다.
- 종료일을 지난 경우 다음 중 하나를 실행한다.
  - 목록에서 제거
  - 상세 상단에 종료 배너 표시
  - 버튼 비활성화
  - 목록 페이지로 이동 유도

## 권장 운영 흐름

1. 팝업 등록
2. 이벤트 게시글 수동 등록
3. 둘 다 같은 식별자로 기록
4. 팝업 종료일 설정
5. 헤더 스크립트가 종료일을 읽어 이벤트 게시글 화면을 제어
6. 종료일 이후 자동으로 종료 상태 표시

## 가능한 것

- 이벤트 게시글을 화면에서 종료처럼 보이게 처리
- 목록 노출을 제어
- 상세 진입 시 종료 안내 표시
- 신청 링크 차단

## 어려운 것

- 아임웹 관리자 DB 자체를 직접 수정
- 게시글을 진짜 비공개로 바꾸기
- 팝업 설정값과 게시글을 내부적으로 자동 동기화
- 게시글을 다른 게시판으로 이동

## 예외 처리

- 종료일이 없는 이벤트는 자동 종료 대상에서 제외한다.
- 이벤트 게시글인데 식별자가 없으면 수동 점검 대상으로 표시한다.
- 공지사항으로 잘못 등록된 건은 자동 처리하지 않는다.

## 제안

- 운영상 가장 안전한 방식은 `팝업 종료일 = 이벤트 종료일`로 강제하는 것이다.
- 이벤트 게시글만 자동 종료 대상으로 두고, 공지사항은 기록용으로 남긴다.
- 실제 종료 상태 변경이 필요해지면 추후 아임웹 API 가능 여부를 다시 확인한다.
