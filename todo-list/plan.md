# 투두리스트 앱 계획서

## 이 문서의 목적

`eunjjang` 폴더 안에 별도 프로젝트로 투두리스트 앱을 만들기 전에, 요구사항과 개발 순서를 한 곳에서 정리한다.

이 문서는 두 가지 역할을 함께 한다.

- 계획 에이전트 지침: 어떤 질문을 하고 어떻게 요구사항을 정리할지 안내
- 프로젝트 계획서: 실제 투두리스트 앱의 기능, 화면, 데이터 구조, 개발 단계 정리

## 계획 진행 방식

- 처음부터 모든 세부사항을 정하지 않아도 된다.
- 가장 중요한 결정부터 1~3개씩 확인한다.
- 답변이 모호하면 예시를 들어 선택지를 좁힌다.
- 첫 버전은 작고 실행 가능한 MVP로 잡는다.
- 계획은 실제 개발자가 바로 구현할 수 있을 만큼 구체적으로 유지한다.

## 먼저 확인할 질문

1. 투두리스트는 웹앱, 모바일앱, 데스크톱앱 중 무엇으로 만들까?
2. 로그인 없이 한 사람만 쓰는 앱인가, 여러 사용자가 계정을 만들어 쓰는 앱인가?
3. 할 일 데이터는 브라우저/기기 안에만 저장해도 되는가, 서버나 데이터베이스가 필요한가?
4. 첫 버전에 꼭 필요한 기능은 무엇인가?
   - 할 일 추가
   - 완료 체크
   - 수정
   - 삭제
   - 마감일
   - 우선순위
   - 카테고리/태그
   - 검색/필터
5. 디자인은 어떤 느낌이어야 하는가?
   - 아주 심플한 메모장형
   - 업무용 대시보드형
   - 귀엽고 가벼운 개인용
   - 모바일 중심

## 현재 목표

`eunjjang` 폴더 안에 별도 프로젝트로 투두리스트 앱을 만든다.

## 초기 가정

아직 세부 답변을 받기 전이므로, 첫 버전은 다음처럼 작게 시작하는 것을 추천한다.

- 형태: 브라우저에서 실행하는 웹앱
- 사용자: 로그인 없는 개인용 앱
- 저장 방식: 브라우저 `localStorage`
- 구현 범위: 할 일 추가, 완료 체크, 수정, 삭제, 필터
- 목적: 빠르게 실행 가능한 MVP를 만든 뒤 필요하면 로그인/서버/DB를 추가

## 핵심 기능 후보

### MVP

- 할 일 추가
- 할 일 목록 보기
- 완료/미완료 체크
- 할 일 내용 수정
- 할 일 삭제
- 전체/진행 중/완료 필터
- 브라우저에 데이터 저장

### 이후 확장

- 마감일
- 우선순위
- 카테고리 또는 태그
- 검색
- 정렬
- 다크 모드
- 통계
- 로그인 및 클라우드 동기화

## 화면 구성 초안

### 메인 화면

- 상단: 앱 이름, 간단한 상태 요약
- 입력 영역: 새 할 일 입력창, 추가 버튼
- 필터 영역: 전체, 진행 중, 완료
- 목록 영역: 할 일 카드 또는 행
- 빈 상태: 할 일이 없을 때 표시할 간단한 문구

## 데이터 구조 초안

```ts
type Todo = {
  id: string;
  title: string;
  completed: boolean;
  createdAt: string;
  updatedAt: string;
};
```

확장 시 추가 가능한 필드:

```ts
type Todo = {
  id: string;
  title: string;
  completed: boolean;
  priority?: "low" | "medium" | "high";
  dueDate?: string;
  tags?: string[];
  createdAt: string;
  updatedAt: string;
};
```

## 추천 개발 계획

1. 프로젝트 형태 결정
   - 순수 HTML/CSS/JavaScript
   - React/Vite
   - Next.js

2. 첫 화면 구현
   - 기본 레이아웃
   - 입력창
   - 목록 영역
   - 빈 상태

3. 상태 관리 구현
   - 할 일 추가
   - 완료 상태 변경
   - 수정
   - 삭제

4. 저장 기능 구현
   - `localStorage` 저장
   - 새로고침 후 복원

5. 필터 기능 구현
   - 전체
   - 진행 중
   - 완료

6. 사용성 정리
   - 입력 검증
   - 키보드 Enter 추가
   - 삭제 확인 또는 실행 취소 검토
   - 모바일 화면 확인

7. 확장 여부 결정
   - 마감일/우선순위/태그 추가 여부
   - 서버 저장이 필요한지 확인

## 아직 결정할 질문

1. 웹앱으로 만들까, 아니면 다른 형태가 필요할까?
2. React/Vite처럼 프레임워크를 쓸까, 아니면 간단한 HTML/CSS/JavaScript로 시작할까?
3. 개인용으로 충분한가, 로그인 기능이 필요할까?
4. 첫 버전에 마감일이나 우선순위가 꼭 들어가야 할까?
5. 디자인은 심플한 업무용, 귀여운 개인용, 모바일 중심 중 어떤 쪽이 좋을까?

## 첫 구현 범위 추천

첫 버전은 `todo-app` 같은 별도 폴더를 만들고, 로그인 없는 개인용 웹앱으로 구현한다.

필수 포함:

- 할 일 추가
- 완료 체크
- 수정
- 삭제
- 전체/진행 중/완료 필터
- `localStorage` 저장

제외하고 나중에 검토:

- 로그인
- 서버 API
- 데이터베이스
- 알림
- 협업 기능
